## 1. E7 总目标

E7 的目标是落地 E 系列**第四个产品能力包**：案件协作工作台。它是闭环的「归集与协作」段——把同一案件场景下散落在各产品的**契约对象**（录入端的 `SearchProfile` 摘要、检索助手的 `CandidateRef`、文书工作台的 `DraftDescriptor`）归集进一个 `CaseFolder`，让律师个人沉淀、让团队在**对象级鉴权 + 默认私有 + 显式共享**下协作，但 casebook **只归集契约对象引用、不复制正文、不起草、不下结论、不输出胜负判断**。

```text
一次案件场景下的契约对象（都来自共享内核 contracts，不 import 产品包）
  SearchProfile 摘要（脱敏白名单子集，来自 intake，原始案情零上送）
  + CandidateRef[]（类案，来自检索/清单，带 source_anchors，零裁判正文）
  + DraftDescriptor[]（文书骨架描述，来自文书工作台，带引用骨架，零起草正文）
  -> 用户在协作台归集进 CaseFolder（默认 visibility=private）
  -> 持久化只存元数据 + 引用 + 短字段 + 多租户字段，零正文
  -> M5 多租户强隔离 + 对象级鉴权：默认私有，显式共享给 team 后团队可见
  -> 导出/汇总走 M4-5 免责头机制：只含元数据 + 来源链接 + 用户备注 + 免责声明
```

E7 的本质是**第四个产品包落地 + 复用 M5 多租户/共享能力做案件级归集协作**，不是放开护栏，也不是让 casebook 直连检索/起草/法条逻辑。E7 不做 E8 闭环反哺（聚合回流），不改排序/召回/source selection/rerank 默认，不改 `/api/search`、E3 `InternalSearchService`、intake、statute、drafting 端点行为，不绕过内核公开面直连检索底层，不 import 其它产品包。

### 1.1 协作工作台与「跨产品聚合」的关键差异（E7 地基判断）

- **只归集契约对象、不 import 产品包**：CaseFolder 聚合的 `SearchProfile` 摘要 / `CandidateRef` / `DraftDescriptor` 都是已冻结在**共享内核 `app.kernel.guardrails.contracts`** 的契约对象类型，casebook 产品包**只依赖内核公开面拿到这些契约类型**，**绝不** import `intake`/`statute`/`drafting` 产品包本身（沿用 E2~E6「产品包互不 import」红线）。聚合 = 持有契约对象引用，不是调用其它产品包的服务。
- **零正文归集、继承上游边界**：归集进 CaseFolder 的每个引用都继承上游白名单与锚点边界——`CandidateRef` 零裁判正文且 100% 有 `source_anchors`、`DraftDescriptor` 零起草正文且 `structure_skeleton` 仅标题、`SearchProfile` 摘要只取脱敏白名单子集（原始案情绝不进入）。无锚点的引用类内容不进 CaseFolder 与导出（沿用 M4「无来源不进交付物」红线）。
- **持久层只存元数据 + 引用 + 短字段**：`CaseFolder` 持久化只存 `case_folder_id`、`owner_user_id`、`team_id`、`visibility`、`search_profile_summary`（脱敏子集）、`candidate_refs`（引用）、`draft_descriptors`（引用）、用户短字段（如 `title`/`note`/`tag`）、时间戳/状态；**绝不存裁判文书正文、候选/chunk 正文、起草正文、原始案情**（沿用 M4 持久层隐私升级 + M5 多租户）。
- **协作即共享、共享即鉴权**：CaseFolder 默认 `visibility=private`；共享 = 把 visibility 从 private 显式改为 team（沿用 M5 共享语义），仅同 team 成员可见；对象级鉴权拦截越权读写（越权返回 404，不泄露存在性）；凭据不代管。

### 1.2 E7 涉及的 flag（沿用 E-1 五产品 flag，不新增产品级 flag）

| flag | 含义 | 默认 | E7 阶段状态 |
| --- | --- | --- | --- |
| `ENABLE_CASEBOOK`（E-1 已冻结） | 协作工作台总开关：前端入口 + casebook 端点是否可用 | `false` | E7 实现其 on 路径（归集 + 持久化 + 共享 + 导出），但默认仍 `false` |
| `VITE_ENABLE_CASEBOOK`（E-1 已冻结） | 前端镜像：协作工作台入口/路由是否渲染 | `false` | 同上，默认仍 `false` |

> E7 **不新增产品级 flag**（与 E5/E6 一致，与 E4 新增 `ENABLE_INTAKE_AI_EXTRACTION` 不同）。协作台明确**不引入任何 AI 归纳/起草/结论开关**——「只归集不起草、不下结论」是结构性红线，不能用 flag 放开。`ENABLE_INTAKE`/`ENABLE_INTAKE_AI_EXTRACTION`/`ENABLE_STATUTE_SEARCH`/`ENABLE_DRAFTING`/`ENABLE_ECOSYSTEM`/`ENABLE_WEIGHTED_RERANK` 等默认全程仍 false。任一默认 `true` 即 `NO_GO`。

### 1.3 E7 守的红线（在 M4/M5 + 生态 3 红线之上）

1. **只归集契约对象、不复制正文、不起草、不下结论**：CaseFolder 只持有 `SearchProfile` 摘要 / `CandidateRef` / `DraftDescriptor` 引用 + 用户短字段；端点/模型不自动生成案件综述正文、不补全结论、不输出胜负概率或诉讼结果判断。
2. **聚合只依赖内核契约、不 import 其它产品包**：casebook 产品包经 `app.kernel` 公开面获得契约对象类型与 sanitize 函数；**绝不** import `intake`/`statute`/`drafting` 产品包；不直连 retrieval/rerank/summary；不复制检索主路径。
3. **引用必带锚点、无锚点不进交付物**：归集/导出引用的 `CandidateRef`/`DraftDescriptor` 内引用必须有 `source_anchors`/`statute_anchors`；无锚点内容降级不进入 CaseFolder 与导出。
4. **CaseFolder 持久层零正文 + M5 多租户隔离 + 对象级鉴权 + 默认私有**：持久化只存元数据 + 引用 + 短字段 + 多租户字段；默认 `visibility=private`；越权读写拒绝；共享 = 显式改 team。出现裁判正文/候选/chunk 正文/起草正文/原始案情型字段，或默认非私有，即 `NO_GO`。
5. **导出强制免责头**：导出复用 M4-5 免责头机制；只含元数据 + 来源链接/锚点 + 用户备注 + 免责声明，不含正文、不含胜负概率、不含确定性法律结论。
6. 沿用 M4/M5 全部红线 + 生态 3 红线：仅锚定来源才展示 / 无来源降级 / 不落正文（裁判文书）/ 不按 case_id 特判 / 不输出胜负概率 / flag 默认 false / 关闭回上一态 / 多租户强隔离 / 凭据不代管；原始案情零上送 / 跨产品只走契约对象 / 回流 100% 聚合无正文（E7 不做反哺，E8 才做）。

- 协作台端点/模型自动生成案件综述正文、补全结论、输出胜负概率或诉讼结果预测。
- `CaseFolder` 持久层或导出落入裁判文书正文 / 候选 / chunk 正文 / 起草正文 / 原始案情。
- 归集/导出引用的 `CandidateRef`/`DraftDescriptor` 内引用缺 `source_anchors`/`statute_anchors`，或无锚点内容进入交付物。
- `CaseFolder` 白名单出现裁判正文型 / 起草正文型 / 原始案情型 / PII 型 / 胜负结论型字段；或未走合同变更就增删 `CaseFolder`/`CandidateRef`/`StatuteRef`/`DraftDescriptor` 字段。
- `casebook` 与其它产品包（intake/statute/drafting）互相 import；或 casebook 不经内核公开面而直接 import 产品包获取契约对象。
- `casebook` 绕过 `app.kernel` 公开面直连 retrieval/rerank/summary，或复制检索主路径，或调用文本生成，或循环导入。
- `CaseFolder` 持久化未做多租户隔离 / 对象级鉴权 / 默认私有；出现 public 级可见性；越权读写未返回 404；非 owner 可写；新增未鉴权对外端点。
- 共享时复制/放开正文；或另起独立于 M5 的第二套权限/共享模型。
- 导出（若有）缺失/弱化免责头，或免责头含个案结论/胜负判断；导出含正文/胜负/确定性法律结论。
- `/api/search`、`/api/search/expand`、E3 `InternalSearchService`、intake 端点、statute 端点、drafting 端点行为无说明地改变；排序、召回、source selection、rerank 默认被改；案件/法条索引或评测基线被改。
- `include_router` ≠ 16（E7-2 后），或共享端点导致计数越界。
- 做了 E8 闭环反哺（聚合回流反哺排序）。
- 任一 E-1/M4/M5/INTAKE/STATUTE/DRAFTING flag、`ENABLE_CASEBOOK` 默认被改为 `true`；或新增任何默认 `true` 的对外业务 flag，或引入任何 AI 归纳/起草开关。

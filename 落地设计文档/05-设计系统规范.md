# 设计系统规范

## 1. 设计原则

类案检索助手面向法律专业场景，视觉和交互要传递克制、可信、可核验，而不是娱乐化或营销化。

1. 专业：界面安静、信息层级明确，避免花哨渐变和过强动效。
2. 高效：结果页为扫读优化，核心事实、相似度、法院和审级必须一眼可见。
3. 可核验：所有 AI 加工内容都能回到原文来源。
4. 不夸大：避免「保证查全」「智能判胜诉」等不可验证表述。
5. 移动可用：375px 宽度下仍能完成输入、查看结果、打开详情。

## 2. 品牌气质

| 维度 | 规范 |
| --- | --- |
| 关键词 | 专业、冷静、可信、快速判断 |
| 避免 | 科幻感、娱乐化、过度 AI 化、浮夸承诺 |
| 视觉参考 | 法律文书的秩序感 + 专业数据库的密度 + 现代搜索工具的轻量交互 |
| 信息密度 | 中等，结果页略高于首页 |

## 3. 颜色 Token

沿用 PRD 的深蓝与琥珀，但扩展中性色、成功色和错误色，避免界面只剩单一蓝色。

```css
:root {
  --color-bg: #f7f9fc;
  --color-surface: #ffffff;
  --color-surface-muted: #f1f5f9;

  --color-text: #1f2937;
  --color-text-muted: #64748b;
  --color-text-subtle: #94a3b8;

  --color-border: #d8e0ea;
  --color-border-strong: #b8c3d1;

  --color-brand: #1a365d;
  --color-brand-hover: #244a7a;
  --color-brand-soft: #e7eef7;

  --color-highlight: #dd6b20;
  --color-highlight-soft: #fff3e8;

  --color-success: #2f855a;
  --color-success-soft: #e8f5ee;

  --color-warning: #b7791f;
  --color-warning-soft: #fff8e6;

  --color-danger: #c53030;
  --color-danger-soft: #fff1f1;
}
```

### 使用规则

| 场景 | 颜色 |
| --- | --- |
| 主按钮、焦点边框 | `--color-brand` |
| 关键事实高亮 | `--color-highlight-soft` 背景 + `--color-highlight` 文本 |
| 低置信度候选 | `--color-warning-soft` |
| 成功/可用 | `--color-success` |
| 网络错误、校验错误 | `--color-danger` |
| 大面积背景 | `--color-bg` 或白色，不使用深色大背景 |

## 4. 字体

| 类型 | 字体 |
| --- | --- |
| 中文 | PingFang SC, Microsoft YaHei, Noto Sans SC |
| 英文/数字 | Geist, system-ui, Segoe UI |
| 等宽数字 | JetBrains Mono, SFMono-Regular, Consolas |

规则：

- 法律软件 UI 禁用衬线字体。
- 正文字号不小于 14px。
- 移动端输入框字号不小于 16px。
- 数字指标如相似度、耗时、排名可使用等宽字体。
- 字间距保持默认，不做负字距。

## 5. 字号层级

| Token | 尺寸 | 用途 |
| --- | --- | --- |
| `text-xs` | 12px | 元数据、标签、辅助说明 |
| `text-sm` | 14px | 次要正文、结果摘要 |
| `text-base` | 16px | 输入内容、正文 |
| `text-lg` | 18px | 结果标题、详情标题 |
| `text-xl` | 20px | 页面小标题 |
| `text-2xl` | 24px | 首页主标题 |
| `text-3xl` | 30px | 仅首页核心表达 |

首页不是营销落地页，不需要超大 Hero 字号。

## 6. 间距与布局

| Token | 值 | 用途 |
| --- | --- | --- |
| `space-1` | 4px | 图标与文字间距 |
| `space-2` | 8px | 表单 label 和控件间距 |
| `space-3` | 12px | 卡片内部小组 |
| `space-4` | 16px | 卡片内边距、列表间距 |
| `space-6` | 24px | 页面区块 |
| `space-8` | 32px | 首页主体 |
| `space-12` | 48px | 大区块间距 |

布局规则：

- 首页搜索框最大宽度 760px。
- 结果页内容最大宽度 1180px。
- 结果卡片固定内部结构，加载、hover、展开不改变卡片宽度。
- 不把页面区块都包成卡片，只对结果项、详情抽屉、弹窗使用卡片或面板样式。

## 7. 圆角、边框、阴影

| Token | 值 | 用途 |
| --- | --- | --- |
| `radius-sm` | 4px | 标签、细小元素 |
| `radius-md` | 8px | 卡片、输入框、按钮 |
| `radius-lg` | 12px | 详情抽屉内部面板 |

规则：

- 结果卡片圆角 8px，符合 PRD。
- 阴影只用于抽屉、浮层和当前 hover 卡片。
- 常规分区用边框、分割线、留白表达，不堆叠卡片。

## 8. 组件规范

### 8.1 Button

| 类型 | 用途 | 样式 |
| --- | --- | --- |
| Primary | 搜索、重试、扩展检索 | 深蓝底白字 |
| Secondary | 查看详情、复制案号 | 白底深蓝字，1px 边框 |
| Ghost | 关闭、清空、返回 | 透明底，hover 浅灰 |
| Danger | 删除草稿、清除历史 | 红色，仅后续版本使用 |

交互：

- hover 只改变背景或边框。
- active 使用轻微 `transform: translateY(1px)`。
- loading 保持按钮宽度，不让布局跳动。
- 图标按钮必须有 tooltip 或 aria-label。

### 8.2 Textarea 搜索框

规格：

- 最小高度 120px。
- 最大高度 260px，超过滚动。
- 右下角显示字数。
- placeholder 给一个完整案情例子。
- 聚焦态边框深蓝。
- 超过 500 字显示弱警示，不阻断提交。

校验：

- 空白不可提交。
- 纯标点不可提交。
- 输入保存到 localStorage 草稿。

### 8.3 ResultCard

结构：

1. 标题。
2. 法院、审级、日期、案由。
3. 2-3 句事实摘要。
4. 高亮事实片段。
5. 相似度条和分数。
6. 查看详情按钮。

视觉：

- 卡片白底，1px 边框。
- 高相似结果不使用夸张色块，只在相似度和高亮上增强。
- 低置信度候选使用更轻的视觉权重。

### 8.4 SimilarityMeter

规则：

- 显示文本：`事实相似度 87%`。
- 分数条最大宽度固定。
- 颜色分段：
  - >= 80：深蓝。
  - 65-79：琥珀。
  - < 65：灰蓝。
- 分数只代表检索相关度，不代表胜诉概率。

### 8.5 HighlightText

规则：

- 高亮背景使用浅琥珀。
- 高亮不超过摘要文本的 35%，避免整段发黄。
- hover 可显示来源片段。
- 不把 AI 生成文本伪装成原文。

### 8.6 CaseDetailDrawer

结构：

- 顶部：标题、案号、关闭。
- 元数据：法院、审级、案由、日期。
- 关键事实摘要。
- 裁判要旨。
- 来源片段。
- 原文链接。

桌面端：

- 右侧抽屉，宽度 440-560px。
- 页面主体保留，便于继续扫结果。

移动端：

- 全屏抽屉。
- 顶部固定关闭按钮和案号。

### 8.7 EmptyResults

空状态文案模板：

```text
未找到足够匹配的案例
可以尝试补充案件经过、损害结果、争议焦点，或简化为最核心的事实动作。
```

提供三个动作：

- 修改描述。
- 使用扩展检索。
- 查看示例案情。

### 8.8 ErrorBanner

规则：

- 网络错误展示在结果区域顶部。
- 错误文案清晰，不暴露技术栈细节。
- 保留已输入内容和已加载结果。
- 必须提供重试按钮。

## 9. 文案规范

### 推荐表达

- 「按事实相似度优先排序」
- 「以下案例与您的案情部分相关，仅供复核」
- 「数据覆盖至 2026-05-01」
- 「自动摘要暂不可用，已展示原文片段」
- 「建议补充行为经过、损害结果或争议焦点」

### 禁止表达

- 「保证无遗漏」
- 「查全率 99%」
- 「胜诉概率」
- 「法官一定会采纳」
- 「AI 已确认」
- 「绝对相同案例」

## 10. 动效规范

| 场景 | 动效 |
| --- | --- |
| 搜索提交 | 按钮 loading，顶部细进度条 |
| 骨架屏 | 浅色 shimmer，1.2s 循环 |
| 卡片 hover | 边框变深，轻微上移 1px |
| 抽屉打开 | transform + opacity，200ms |
| 错误出现 | fade in，避免弹跳 |

性能规则：

- 只动画 transform 和 opacity。
- 不动画 width、height、top、left。
- 骨架屏尺寸必须与真实卡片一致。
- 动效不影响阅读。

## 11. 响应式规范

### 移动端

- 搜索框全宽。
- 结果卡片单列。
- 相似度放在标题下方，避免右侧挤压。
- 详情抽屉全屏。
- 按钮高度至少 40px。

### 桌面端

- 搜索结果列表宽度 680-760px。
- 详情抽屉可以和列表并排。
- 顶部搜索条 sticky。
- 示例案情可以横向排列，但不要做三张等宽营销卡。

## 12. 可访问性规范

- 所有表单控件必须有 label。
- 错误消息与输入框通过 `aria-describedby` 关联。
- 抽屉打开时锁定背景滚动。
- 颜色不是唯一信息表达方式。
- 文本对比度符合 WCAG AA。

## 13. 设计验收清单

- 首页第一屏能直接开始检索，不是营销页。
- 375px 宽度下文本不溢出按钮和卡片。
- 结果卡片无需打开详情即可判断大致相关性。
- 低置信度候选和主结果视觉上明确区分。
- AI 摘要、原文片段、来源引用有清晰边界。
- 所有加载、空、错、降级状态都有设计稿或实现样式。
- 视觉不依赖单一蓝色，关键高亮和风险提示有独立语义色。

## 端点契约

所有端点 `prefix=/api/casebook`，均需登录，均受 `ENABLE_CASEBOOK` 门控（默认 false → 403 `CASEBOOK_DISABLED` 安全降级）：

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/folders` | 创建 CaseFolder（默认 `visibility=private`），出已收敛对象零正文 |
| GET | `/folders` | 列出当前用户/团队可见 CaseFolder（对象级鉴权 + 租户隔离 + visibility 过滤） |
| GET | `/folders/{case_folder_id}` | 读取单个（越权 404） |
| PUT | `/folders/{case_folder_id}` | 更新归集引用/短字段（仍只存元数据，经 sanitize；非 owner 404；本步可写 visibility） |

请求体 `extra=forbid`（第一道闸）+ service `sanitize_case_folder`（第二道闸）。
错误码：`CASEBOOK_DISABLED` / `CASE_FOLDER_REJECTED` / `CASEBOOK_REQUIRES_LOGIN` / `CASE_FOLDER_NOT_FOUND`。

## 持久层列清单（证明无正文列）

`case_folder` 表 14 列，全为元数据 / 多租户 / 引用(JSON) / 短字段：

`case_folder_id`(PK) / `owner_user_id`(index) / `team_id`(index,nullable) /
`visibility`(default private) / `search_profile_summary`(JSON,脱敏白名单子集,nullable) /
`candidate_refs`(JSON,带锚点引用) / `draft_descriptors`(JSON,带锚点引用) /
`title` / `note` / `tag`（短字段） / `status` / `reason_code` / `created_at` / `updated_at`。

无裁判正文 / 起草正文 / 候选 / chunk 正文 / 案件综述 / 原始案情 / 凭据列
（`models.py` grep：`NO_BODY_COLUMNS`，无 `password/token/secret/credential` 列）。

## 归集而非起草的实现方式

`service` 内无任何 LLM / 模型 / 检索客户端 import；`assemble_case_folder()` 只调用
内核 `sanitize_case_folder` 做白名单清洗 + 锚点收敛，**绝不生成任何案件综述正文 / 结论 /
胜负判断 / 诉讼结果预测**。`candidate_refs` / `draft_descriptors` 逐项收敛，缺锚点 fail-closed
丢弃；`search_profile_summary` 收敛为 SearchProfile 脱敏白名单子集。出库再经一次
`sanitize_case_folder`（双保险）。

## casebook 只 import 内核契约不 import 产品包的证明

AST 扫描 `app/casebook/*.py` 全部 import：

- 禁止产品包 import：`[]`（无 intake/statute/drafting）
- 禁止检索底层 import：`[]`（无 retrieval/rerank/summary/query_processing）
- 深引内核私有：`[]`
- 实际内核 import：`app.kernel.guardrails`、`app.kernel.identity`（仅顶层公开面）

契约类型 `CaseFolder` / `sanitize_case_folder` 经 `app.kernel.guardrails` 公开面取，
绝不 `from app.intake` / `from app.drafting` import。

## 鉴权 / 租户隔离 / 降级

- 行级隔离 `_tenant_clause`：单用户私有态（owner + team IS NULL）/ 团队态（owner 私有 OR team+visibility=team），与 `DraftDescriptorRow` 同口径。
- 对象级鉴权：`update_owned` 按 `owner_user_id` 取行，非 owner 取不到 → None → 404；读取无「无过滤」路径。
- 默认 `visibility=private`（创建恒 private；PUT 可写但须满足租户一致性，team 可见性要求团队上下文）。
- 跨用户：列表为空 / 读取 404 / 更新 404，不泄露他人 folder 存在性。
- 降级：`ENABLE_CASEBOOK=false` → 全端点 403；flag 开启但未登录 → 401。

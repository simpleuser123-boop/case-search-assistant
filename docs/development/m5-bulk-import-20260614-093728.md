- 落地：M4-7 评估中 `bulk_import: not_ready` 的实现。

## 1. 目标与基调

支持把既有清单/案例批量导入团队空间，便于律所把历史工作产物迁入协作环境。核心红线与 M5-1~M5-5 一脉相承：

- **导入即元数据/引用 only，绝不引入正文或原始案情。**
- 只走白名单字段；非白名单字段（含潜在正文/凭据）一律丢弃或整项拒绝。
- 沿用 `case_id` + `source_chunk_id` 做锚点完整性校验；缺锚点项降级或拒绝，**绝不伪造锚点**。
- 导入对象默认归属当前 owner、默认私有（`team_id=None` / `visibility=private`）；可见性仍由 M5-3 `tenant_visibility_clause` 唯一承载。
- `ENABLE_BULK_IMPORT` 默认 false；关闭即回 M5-5/M4 末态，不提供导入入口。

## 2. 后端实现

新增包 `apps/api/app/bulk_import/`：

| 文件 | 职责 |
| --- | --- |
| `models.py` | `BulkImportJob` 作业账本（表 `m5_bulk_import_job`）：仅聚合统计 + 短状态码 + 结构化关系 + 时间戳，无正文列、无导入项明细 |
| `validation.py` | 纯逻辑：白名单过滤 + 黑名单拦截 + 锚点完整性 + 锚点净化（只留 `case_id`/`source_chunk_id`/`anchor_type`） |
| `store.py` | 作业账本读写 + owner 私有域 `case_id` 去重查询；不写正文，不放权读取沉淀对象 |
| `service.py` | 编排：逐项校验净化 → 按 `case_id` 去重 → 写 owner 私有沉淀（复用 M5-3 `create_sediment`）→ 记作业账本 |
| `schemas.py` | `ImportItem` / `BulkImportRequest` 均 `extra=forbid`，请求边界第一道拦截正文键 |
| `api/bulk_import.py` | 路由 `/api/bulk-import/run` + `/jobs`，flag-gated + login-required |

`main.py` 仅新增 `import` 与 `include_router(bulk_import_router)` 两行。

### bulk_import_job 数据结构

`import_job_id` / `source_type`（`case_list_file`/`csv`/`existing_list`）/ `item_count` / `imported_count` / `rejected_count` / `duplicate_count` / `import_status`（completed/partial/rejected/failed）/ `degrade_reason` / `owner_user_id` / `team_id` / `created_at` / `updated_at`。

> 设计取舍：导入项明细**不入账本表**，避免正文经明细列混入；账本只保留聚合计数与短状态码供审计/进度展示。实际导入对象写入 M5-3 `m5_sedimentation_object`（默认 owner 私有）。

### 导入内容边界

- 白名单：`case_id` / `case_number` / `court` / `trial_level` / `case_cause` / `judgment_date` / `source_anchors` / `note` / `tag` / `label` / `list_id` / `list_title` / `report_id`。
- 黑名单（命中即整项拒绝）：`raw_query` / `query` / `case_fact_body` / `candidate_body` / `chunk_body` / `judgment_long_text` / `summary_body` / `holding_body` / `compare_body` / `user_free_long_text` / `text` / `content` / `password` / `token` / `session_token`。
- 锚点规则：AI 内容承载型（`case_list` / `report_template`）无合法锚点则拒绝；任何出现的锚点都必须合法（`case_id` + `source_chunk_id` 均非空），否则拒绝；锚点净化只保留三个白名单键。纯引用型（`case_favorite`）可不带锚点。
- 去重：按 `case_id` 去重（批内 + owner 私有域已存在）。
- 归属：导入对象默认 owner 私有；`team_id` 仅进作业账本留痕，**不据此放权可见性**。

### 降级与兼容

- flag 关闭 → 403 `BULK_IMPORT_DISABLED`，不建表/不导入。
- 未登录 → 401 `BULK_IMPORT_REQUIRES_LOGIN`。
- 请求体带正文键 → 422（schema `extra=forbid`）。
- 作业级失败（空批 / 非法来源 / 超大批 >500）→ 写 `failed` 账本留痕，安全返回，不影响主链路。

### 正文泄露抽查（实际落库结果）

构造脏批（含 `case_fact_body` / `raw_query` / `chunk_body` / 锚点夹带 `judgment_long_text` / 未知列 / `password`）导入后抽查实际落库行：

- 含正文/凭据键的项被**整项拒绝**（`forbidden_body_field`），不静默吞掉正文后继续。
- 仅干净锚点项落库，锚点被剥离为 whitelist-only。
- leak terms（案情正文/原始查询/判决书长文本/锚点夹带正文/secret123/未知列）命中 **0**。
- 判定：**PASS**。

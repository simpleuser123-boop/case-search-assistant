## 风险提示边界

- 风险提示只表达“建议复核”“可能需要关注”“供复核”。
- 风险提示只作为复核线索，不输出法律结论，不输出胜诉概率或败诉概率，不给出确定性判断或具体诉讼建议。
- 风险提示必须绑定来源锚点；无来源锚点的风险提示不生成或不展示。
- 风险提示不影响主结果排序、低置信度候选分层、rerank 或 source selection。
- 风险提示生成不使用 qrels、label、relevance、query id 或 case id 特判。
- 如果无法可靠生成有来源风险提示，返回空列表和安全状态。

## risk_hint schema

```text
risk_hint:
  risk_type: risk_type enum
  source_anchors:
    - case_id: string
      source_chunk_id: string
      chunk_type: string | null
      anchor_type: string
      source_url: string | null
      source_ref: string | null
  confidence_level: high | medium | low
  confidence_reasons: reason_code[]
  reason_code: reason_code
  review_note: safe fixed review note | null
```

## risk_type 定义

| risk_type | 口径 |
| --- | --- |
| `fact_difference` | 来源片段与当前案情可能存在事实差异，供复核。 |
| `key_element_missing` | 候选命中的关键要素较少，供复核。 |
| `low_confidence_candidate` | 低置信度候选，仅作为补充复核线索。 |
| `adverse_tendency_source` | 来源片段可能包含不利倾向线索，供复核。 |
| `degraded_or_uncertain` | 降级或不确定状态下的来源复核提醒。 |

## 来源锚点要求

- `source_anchors` 必须存在。
- 每个展示的风险提示至少有一个可用来源锚点。
- 可用来源锚点必须能追溯到 `case_id` 和 `source_chunk_id`。
- 后端 `build_risk_hints` 只从已可见的主结果和低置信度候选读取来源锚点。
- 后端 `_valid_source_anchors` 过滤缺少 `case_id` 或 `source_chunk_id` 的锚点。
- 前端 `RiskHintsPanel` 再次过滤无来源锚点风险提示。
- 不创建伪来源、伪锚点或模糊来源。

## 无来源风险提示降级策略

| 场景 | 策略 |
| --- | --- |
| 候选没有可用 `source_anchors` | 后端不生成风险提示。 |
| 降级原因存在但没有任何可锚定结果 | 后端返回空 `risk_hints`。 |
| API 返回无来源风险提示 | 前端过滤，不展示风险提示区。 |
| 风险提示无法可靠生成 | 返回空列表，主检索结果继续可用。 |

## 前端展示策略

- 新增独立 `RiskHintsPanel`，展示在结果概览之后、主结果列表之前。
- 区域文案明确“供复核”。
- 文案说明风险提示只是不影响主结果排序的来源锚点复核线索。
- 每条风险提示展示 `risk_type` 标签、脱敏 `reason_code`、`case_id` 和 `source_chunk_id`。
- 点击“查看来源”打开既有案例详情入口，用于查看来源锚点。
- 不使用后端 `review_note` 作为自由正文展示，前端使用固定安全文案。
- 当风险提示加载失败或被过滤为空时，不影响主结果展示。

## risk_hints 不影响排序的证据

- `apps/api/app/api/search.py` 先执行检索、rerank 和低置信度分层。
- 主结果 `results` 和 `low_confidence_candidates` 生成完成后，才调用 `build_risk_hints`。
- `build_risk_hints` 只返回 `risk_hints` 字段，不返回排序分数、rank override 或 source selection 指令。
- 搜索完成日志只记录 `risk_hint_count`，不记录风险提示正文。
- `apps/api/tests/test_m2_risk_hints.py` 覆盖 risk_hints 不改变 `results` 和 `low_confidence_candidates` 顺序。
- `ENABLE_WEIGHTED_RERANK=false` 在 `apps/api/app/core/config.py` 和 `.env.example` 中继续成立。

## qrels / label / relevance 边界

- 后端风险提示生成过滤 metadata 中的 `qrels`、`label`、`relevance`、`query_id`、`eval_query_id`、`case_id`、`candidate_case_id`。
- focused tests 覆盖 qrels、label、relevance、query id、case id 改变时风险提示签名不变。
- 本步骤未修改 qrels、label 或历史评测结果。
- 本步骤未改变线上排序、source selection 或 rerank 默认开关。

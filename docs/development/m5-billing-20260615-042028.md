## 1. 目标与基调

面向律所采购落地「套餐 + 试用 + 计费 + 续费意愿」闭环。核心红线：
**工具内绝不代填、代管、代存任何支付凭据**（卡号 / 银行账户 / CVV / 第三方支付令牌明文）；
支付由平台侧 / 第三方完成，工具仅记录脱敏回执引用（payment_ref 单向哈希 + status）。
计费状态不改变检索质量与主排序；`ENABLE_BILLING` 默认 false，关闭即回 M5-8 末态。

## 2. 后端改动（apps/api/app/billing/ 新增包 + api/billing.py）

- `privacy.py`：支付凭据护栏 `assert_billing_output_clean`（fail-closed）。
  - 禁止键集：卡号/银行账户/CVV/各类支付令牌明文 + 正文/原始 query 键。
  - 值级兜底正则：13~19 位连续数字（边界排除字母/数字/下划线，避免 UUID/案号/金额误伤）疑似卡号；
    `tok_live_/sk_live_/pi_/card_…` 等第三方令牌前缀字面量。
  - 经测试：7 类凭据样本全部拦截；订阅 ID/案号/金额/回执哈希等良性值零误报。
- `models.py`：三张表，**仅白名单字段，无任何凭据列**。
  - `m5_billing_plan`：套餐目录（套餐名/额度展示/价格展示/周期/席位/试用天数/解锁能力/是否在售）。价格仅展示。
  - `m5_subscription`：订阅 + 试用 + 续费意愿（套餐引用/trial_status/subscription_status/renewal_intent/自填短理由/归属 owner+team/时间戳/reason_code）。
  - `m5_payment_receipt_ref`：支付回执**脱敏引用**（payment_ref 单向哈希 + 结算状态 + 金额展示串 + reason_code）。
- `schemas.py`：所有入参 `extra=forbid`；塞入凭据键直接 422。回执端点只收 `payment_ref`（service 层立即哈希）。
- `store.py`：每次写入前 `assert_billing_output_clean`；回执只落哈希，原始回执号绝不入库；提供 owner 级隔离读取。
- `service.py`：种子化默认套餐（plan_solo/plan_team_pro/plan_firm）；试用开通/到期状态机；续费意愿采集；回执联动订阅状态（仅状态机，不扣款）。脱敏埋点事件 `BillingAnalyticsEvent` 仅 plan_id/status/reason_code/count + subscription_id_hash。**不 import 检索/rerank/retrieval**。
- `api/billing.py`：`/api/billing/{plans,trial,renewal-intent,payment-receipt,subscription}`，flag 双闸（关闭 403 BILLING_DISABLED）+ 需登录（401）。日志只记哈希/枚举/计数。已注册进 main.py。

## 3. 前端改动（apps/web/src）

- `config/featureFlags.ts`：新增 `isBillingEnabled()`，默认 false。
- `services/billingApi.ts`：只读/写脱敏字段，**无任何卡号/CVV/银行账户/令牌字段**；403→disabled、401→login_required、400→guard_block 分类。
- `components/billing/BillingPanel.tsx`：flag 关闭渲染 null（回 M5-8 末态）；展示套餐目录、开通试用、采集续费意愿、脱敏订阅状态；**无任何凭据输入框**；常驻提示「不收集任何卡号/银行账户/CVV/支付凭据，请勿输入」。
- `.env.example`：补 `VITE_ENABLE_TENDENCY_ANALYSIS=false`、`VITE_ENABLE_BILLING=false`。

## 2. 实现要点

### 2.1 Feature flag（默认安全态）

- 前端开关 `VITE_ENABLE_CASE_FAVORITE`，读取器 `apps/web/src/config/featureFlags.ts:isCaseFavoriteEnabled()`，默认 `false`。
- 关闭时：不读写任何浏览器本地存储、不渲染收藏按钮 / 收藏列表，页面回到 M4-2 末态，标准搜索与排序默认行为不变。
- 后端 `ENABLE_CASE_FAVORITE` 续 `false`（config.py 未改动）。

### 2.2 数据结构 `case_favorite`（local-only、可清除）

`apps/web/src/lib/caseFavorite.ts`，storageKey=`case-search:m4:case-favorite:v1`，最多 200 条，按 `created_at` 倒序、按 `case_id` 去重。每条记录字段（全部白名单内）：

- `case_id`：案例稳定标识，去重与回跳详情用。
- `case_number`：案号（元数据）。
- `court` / `trial_level` / `case_cause` / `judgment_date`：案例元数据。
- `source_anchors`：可选，回跳详情用；仅含 `case_id`+`source_chunk_id`(+`anchor_type`/`chunk_type` 短枚举)，**无正文**；只取归属本案的锚点，最多 4 条。
- `note` / `tag`：用户自填短字段，分别截断到 120 / 24 可视字符。
- `created_at`：收藏时间戳。
- `favorite_status`：恒为 `favorited`（取消即从列表移除）。

构造器 `buildFavoriteRecord` 只从已在屏的元数据来源（`SearchResultItem` / `CaseDetailResponse`）读取上述字段，**不读取** summary / matched_text / chunk 正文 / 裁判文书正文。反序列化 `loadFavorites` 时 `sanitizeRecord` 仅重建白名单键，主动丢弃任何旧版本 / 篡改数据混入的非白名单键（含潜在正文字段）。

### 2.3 收藏边界

- 只保存白名单内的元数据、来源引用和用户自填短字段。
- 不复制并存储案例正文、摘要正文、chunk 正文。
- 收藏可增（`addFavorite`，幂等去重）、可删（`removeFavorite`）、可清空（`clearFavorites`）、可改短字段（`updateFavoriteFields`）。
- 本地存储异常（隐私模式 / 配额 / JSON 损坏）一律安全降级，绝不破坏主检索 / 阅读链路。

### 2.4 展示规则

- 结果卡片（`ResultCard`，主结果 + 低置信候选）、案例详情抽屉（`CaseDetailDrawer` 元数据区）、对比视图每案 chip（`CaseCompareView`）均可收藏 / 取消收藏，复用同一 `FavoriteButton`。
- 收藏列表（`FavoritesPanel`，侧栏）展示案号 + 元数据 + 自填标签 / 备注 + 收藏时间，提供「查看详情」回跳、编辑 note/tag、取消单条、清空全部（二次确认）。
- 回跳详情用收藏记录里的 `case_id` + 元数据 + 锚点构造最小 seed（无正文），交给与结果卡片一致的详情抽屉打开。

### 2.5 隐私与排序规则

- 收藏不进入主排序、召回或 source selection；`getFavoriteSelection` 只读 `case_id` 是否已收藏，不回写任何排序特征。不按 case id / qrels / label / relevance 特判。
- 脱敏日志 `case_favorite_action` 只记录 `event`/`surface`/`status`/`reason_code`/`count`，不记录案号、note、tag、query 或任何正文。
- 收藏正文边界：本地存储 blob 经断言验证不含「本院查明 / 裁判要旨 / 摘要」等正文；`/api/events` 请求体经断言验证不含原始案情正文与案号。

## 1. 目标与基调

引入服务端用户身份与认证最小骨架，把当前匿名单用户态映射到具名 `user_id`，为后续团队隔离、权限、共享提供身份基础。沿用「先合同 + 骨架，敏感项谨慎落地」：本步只做账号/认证最小骨架；密码仅单向哈希、会话令牌仅存哈希；`ENABLE_ACCOUNT_SYSTEM` 默认 `false`，关闭后行为与 M4 单用户私有完全一致。

凭据类敏感输入由用户/平台侧完成：工具不代填、不代管、不代存明文；SSO/OAuth 令牌不落业务表、不写日志。

## 2. 账号数据结构（仅白名单字段 + 密码哈希列）

表 `m5_account` 仅含合同声明的白名单列：`user_id`、`login_name`、`display_name`、`account_status`、`auth_provider`、`auth_subject_ref`、`password_hash`、`created_at`、`updated_at`、`reason_code`。

- `display_name` / `login_name` 为用户自填短字段（长度受限，防自由长文本混入）。
- `password_hash` 只存单向哈希（local 账号）；SSO 账号为空。
- `auth_subject_ref` 只存 SSO 的 `provider:subject_id` 引用，绝不存令牌明文。
- 经断言确认表中**不存在** `plaintext_password` / `password` / `token` / `session_token` / `raw_query` / `case_fact_body` 等列。

## 3. 密码哈希（不可逆、无明文）

模块 `apps/api/app/core/password.py` 采用标准库 `pbkdf2_hmac`（SHA-256，默认 240000 次迭代）+ 每用户随机盐，存储格式自描述：`pbkdf2_sha256$<iterations>$<salt_hex>$<hash_hex>`。校验用常量时间比较。算法可平滑替换为 bcrypt / argon2。存储层 `AccountStore.create_account` 在写入前断言 `password_hash` 已是哈希格式，任何明文直接 `ValueError` 拦截，绝不入库。

## 4. 认证骨架（注册 / 登录 / 登出 / 会话）

路由 `apps/api/app/api/auth.py`，前缀 `/api/auth`：

- `POST /register`：创建账号，**不下发** session_token。
- `POST /login`：校验密码哈希，成功生成原始会话令牌**一次性**下发给客户端。
- `POST /logout`：按令牌哈希吊销会话。
- `GET /session`：按令牌哈希解析当前会话，仅返回公开账号视图。
- `POST /claim`：单用户态迁移认领（需显式 `confirm=true`）。

会话表 `m5_account_session` 只存 `token_hash`（sha256 hex），原始令牌不进业务表、不进日志、不进报告/JSON/测试快照。登出即将会话置为 `revoked`，复用旧令牌的 `/session` 返回 `ok=false`。

## 5. 匿名态 / 关闭态兼容

`ENABLE_ACCOUNT_SYSTEM` 默认 `false`。关闭态下：

- 所有 `/api/auth/*` 端点返回 `403 ACCOUNT_SYSTEM_DISABLED`，不建表、不读写、零副作用（懒初始化，仅 flag 打开时才建表/构造服务）。
- 前端 `AccountPanel` 在 `isAccountSystemEnabled()=false` 时直接渲染 `null`，不暴露任何登录/注册入口。
- 沉淀仍走本地 `localStorage`，与 M4 单用户私有末态完全一致。

## 6. 单用户态迁移认领路径

模块 `apps/api/app/account/migration.py` + 端点 `/api/auth/claim`：

- **默认不自动执行**：必须显式 `confirm=true` 才评估；否则返回 `confirm_required`。
- 仅元数据 / 引用 / 来源锚点 / 用户自填短字段进入认领；含正文/凭据键即整项 `rejected`，未知键 `rejected`。
- 锚点继承 `case_id + source_chunk_id`：缺锚点项 `degraded`（不伪造 `source_chunk_id`）。
- 输出仅脱敏计数与 reason_code 分布；`owner_user_id` 仅以 `uidh_` 哈希出现，无明文 user_id、无被认领项正文。
- M5-2 只做「校验 + 计数 + 归属标记」纯函数骨架，不把对象写进任何团队/服务端业务表（留待 M5-3 之后）。

## 7. 隐私与凭据规则

- 后端日志只记录 `user_id_hash`（`uidh_` 前缀，确定性、脱敏）/ `ok` / `reason_code` / `path` / 计数；经断言确认**不**记录 `login_name`、`display_name`、`password`、`session_token`。
- 密码、令牌、SSO 凭据不写入日志、报告、JSON 或测试快照。
- 工具不代填密码 / SSO 凭据：凭据值全部来自用户在平台侧的输入。

## 8. 前端（flag-gated）

- `apps/web/src/lib/sessionState.ts`：会话令牌只存**内存运行态**，绝不写 `localStorage` / `sessionStorage` / JS cookie；刷新即丢失（需重新登录）。公开账号视图仅 `user_id` / `display_name` / `account_status` / `auth_provider`。
- `apps/web/src/services/authApi.ts`：只把表单输入的凭据一次性发送，不缓存、不持久化密码；登录返回的令牌交给 sessionState（仅内存）；后端 403 映射为 `disabled`（回到 M4 匿名态）。
- `apps/web/src/components/account/AccountPanel.tsx`：flag 关闭渲染 `null`；提交后立即清空密码字段；未接入默认页面（仅提供组件）。
- `apps/web/src/config/featureFlags.ts` 新增 `isAccountSystemEnabled()`（默认 false）。

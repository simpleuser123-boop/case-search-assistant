## 3. 契约字段集（冻结口径）

intake 只产出 `SearchProfile`（白名单五字段），只消费 `CandidateRef`（白名单七字段）。两者均 `is` 复用 E-1 冻结 frozenset，不另写第二套白名单。

SearchProfile 白名单（5 字段，= E-1 / 文档 17 §3）：

    case_cause, region, trial_level_preference, dispute_focus_keywords, query_text

CandidateRef 白名单（7 字段，= E-1 / 文档 17 §3，intake 只读不重构）：

    case_id, case_number, court, trial_level, case_cause, judgment_date, source_anchors

PII 型键黑名单（录入端特有红线 `INTAKE_FORBIDDEN_PII_KEYS`，5 组）：姓名/身份（name/full_name/party_name/...）、证件号（id_card/passport_no/social_credit_code/...）、联系方式（phone/mobile/email/...）、金融（bank_card/bank_account/...）、地址/车牌（address/residence/plate_no/...）。仅冻结**键名**口径；值层面脱敏（移除 PII token）属 E4-2 范畴。

## 4. 原始案情零上送断言口径

`assert_no_raw_case_payload(payload)`（纯函数，fail-closed）：

- 任何 intake 入参 / 模型出现**正文型键**（复用 E-1 `FORBIDDEN_BODY_KEYS`：raw_case/raw_query/full_text/content/chunk_text/...）→ 抛 `ContractViolationError`。
- 出现**PII 型键**（`INTAKE_FORBIDDEN_PII_KEYS`：name/id_card/phone/address/email/...）→ 抛 `ContractViolationError`。
- 异常消息**只暴露键名、绝不回显键值**——避免原始 PII/正文经异常或日志外泄（已有单测断言 secret value 不出现在异常字符串中）。
- `sanitize_intake_search_profile` 先跑该断言，再只保留 SearchProfile 白名单五字段、主动丢弃其余键。

红线层级：正文/PII 出现在 intake 入口 = NO_GO 级事件，必须显式失败，不静默丢弃。

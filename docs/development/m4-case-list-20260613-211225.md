## 三、组装规则

- 来源入口：结果列表（含低置信候选）、案例对比视图（compare）、详情抽屉均可「加入清单」。
- 清单内可手动排序（仅影响清单展示）、可增删项、可删除整张清单；同一案例同一清单去重。
- 加入清单时只从屏上已有的元数据来源构造清单项（元数据 + 锚点 + 用户自填短字段），绝不写入任何正文。

## 四、来源规则

- 每个清单项引用的案例侧内容必须可追溯 `case_id + source_chunk_id`；`sanitizeListAnchors` 丢弃跨案锚点与缺字段锚点。
- 无锚点的项在面板内降级展示「来源引用暂缺」，不伪造 `source_chunk_id`。

## 五、边界规则（与止损线对应）

- 清单只保存引用与用户自填字段，不复制正文（持久层 + 内存态均无正文）。
- 清单不作为运行时排序、召回或 source selection 的输入；`getListSelection` 只读「本案在哪些清单内」的引用关系，绝不回写排序特征。
- 用户手动排序（reorder/move）只改 `items` 展示顺序，绝不回写主结果排序。
- 不按 query id / case id / qrels / label / relevance 做清单特判；不把低置信候选包装成高置信主结果再入清单（候选仍带「部分相关，仅供复核」标识，入清单只复制元数据与锚点）。

## 2. 本步交付物

### 2.1 共享平台三层分层契约（冻结为文档与产物，零实现）

冻结「前端层 / 产品能力包层 / 共享内核层」三层边界与依赖方向：

- 产品能力包**只能依赖共享内核**，不能互相 import；跨产品数据流转只走契约对象，不走代码直连。
- 护栏（来源锚点校验、字段白名单 sanitize、禁用文案守门、多租户过滤）全部在共享内核实现，能力包调用不重写。
- 每个产品一个总开关 flag，默认 false；关闭某产品 = 前端入口不渲染 + API 端点安全降级，其余产品与基础搜索不受影响。

本步只把上述写成文档与机器可校验产物，**不抽内核、不暴露检索服务、不建产品包**（分别属 E-2 / E-3 / E-4~E-7）。

### 2.2 四个跨产品契约对象字段白名单（冻结，与文档 17 §3 逐字段一致）

实现于 `apps/api/app/contracts/whitelist.py`（仅常量 + sanitize 纯函数）：

| 契约对象 | 流向 | 字段集 |
| --- | --- | --- |
| SearchProfile | 录入端 → 检索助手 | case_cause / region / trial_level_preference / dispute_focus_keywords / query_text |
| CandidateRef | 检索助手 → 法条/文书/协作台 | case_id / case_number / court / trial_level / case_cause / judgment_date / source_anchors |
| DraftDescriptor | 文书工作台 → 协作台 | draft_id / structure_skeleton / candidate_refs / note / tag |
| CaseFolder | 协作台内归集 | case_folder_id / owner_user_id / team_id / visibility / search_profile_summary / candidate_refs / draft_descriptors / created_at / updated_at |

- 四个字段集与文档 17 §3.1~§3.4 **逐字段一致**（focused 测试硬编码断言）。
- 无任一白名单含正文型字段（与 FORBIDDEN_BODY_KEYS 黑名单不相交，测试断言）。
- `sanitize_contract` 为纯函数：丢弃非白名单键；遇正文型键 fail-closed 抛 `ContractViolationError`（不静默丢弃）。
- 模块**不 import** retrieval/rerank/summary/query_processing/case_store 任何内核运行时；**未被任何现有端点接线**。

### 2.4 生态级隐私红线（冻结为可校验断言口径）

1. **录入端原始案情零上送**：`user_raw_case_persisted_on_server=false`（沿用 M4-1）；原始口语化案情仅浏览器本地。
2. **跨产品传递只走契约对象**：产品间不直接共享内存对象或数据库表，只通过 4 个契约对象传递，每个过白名单 sanitize。
3. **回流聚合无正文**：反哺数据 100% 聚合/规则，抽查 bodyHits 必须为空、rawQueryLeak 必须为 false。

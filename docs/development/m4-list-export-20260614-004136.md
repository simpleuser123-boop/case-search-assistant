## 一、做了什么

支持把一张类案清单导出为 **Markdown / CSV** 文件，在浏览器本地生成并下载，便于线下整理与交付。导出文件**只含元数据、来源引用与用户自填备注**，强制带数据覆盖与免责说明，绝不含裁判文书正文，绝不含胜负概率或确定性法律结论。

## 二、case_list_export 数据结构

```
CaseListExportDescriptor {
  export_id        // 本地生成，无语义
  list_id          // 引用，不含正文
  export_format    // "markdown" | "csv"
  fields           // 导出列，限白名单子集
  export_status    // "exported" | "empty" | "degraded" | "failed"
  degrade_reason   // ExportReasonCode | null
  item_count       // 仅计数
  created_at
}
```

## 三、导出内容边界（白名单）

导出列严格限定为 8 个白名单字段，与 M4-1 合同「允许持久化 / 可导出」字段集一致：

`case_number`（案号）、`court`（法院）、`trial_level`（审级）、`case_cause`（案由）、`judgment_date`（裁判日期）、`source_anchor`（来源引用 = case_id#chunk_id）、`note`（用户备注）、`tag`（用户标签）。

**绝不导出**：裁判文书正文 `judgment_long_text`、案情正文 `case_fact_body`、候选正文 `candidate_body`、chunk 正文 `chunk_body`、摘要 / 要旨 / 对比正文、原始 `raw_query`、任何自由长文本。

`exportFieldValue` 用 switch 白名单取值，未知列返回空串；`resolveExportFields` 过滤非白名单列、去重、保序，空输入回退全列。即使来源对象被塞入 body 型键，也只读白名单字段。

## 四、免责 / 数据覆盖说明（强制文件头部）

每个导出文件头部强制写入 4 行说明（Markdown 用引用块 `>`，CSV 用 `# ` 注释行）：

1. 仅含元数据 / 来源引用 / 用户备注，不含裁判文书正文或原始案情；
2. 数据覆盖说明：阶段性结果，可能存在未覆盖案例，不代表完整检索；
3. 不对完整性作承诺，不提供胜诉 / 败诉等诉讼结果判断及确定性法律结论；
4. 使用前请结合权威数据库与原始裁判文书人工复核。

措辞规避「已查全 / 保证无遗漏 / 查全率 / 胜负概率」等绝对话术。运行时自检 `containsForbiddenExportPhrase` 覆盖 11 个禁用词，免责文案自身通过该校验。

## 五、导出方式与安全降级

- 浏览器本地：`Blob` + `a[download]` + `objectURL` 生成下载，无后端往返。
- 注入式下载器（`DownloaderLike`）：生产用浏览器实现，测试用内存实现。
- 降级路径：
  - 缺清单 → `failed` / `list_not_found`；
  - 空清单 → `empty` / `empty_list`，不下载；
  - 环境不支持下载（无 Blob/URL）→ `degraded` / `download_unavailable`，**内容已生成可复核**；
  - 下载器抛错 → `degraded` / `download_failed`。
- 所有路径都包 try/catch，**绝不抛出、绝不破坏主检索 / 阅读 / 清单链路**。UI 给出安全提示。

## 六、隐私与文案规则

- 埋点 `case_list_export` 只记录 `event / format / status / reason_code / count`，**绝不含正文、案号、note、tag、title、query**。实测日志：`{"event":"case_list_export","format":"csv","status":"exported","reason_code":null,"count":1}`。
- 导出不参与主排序 / 召回 / source selection；`ENABLE_WEIGHTED_RERANK` 续 false。
- flag 默认 false：关闭时不渲染任何导出入口、不生成文件，页面回到 M4-4 末态。

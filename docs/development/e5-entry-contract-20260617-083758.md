## 3. StatuteRef 字段集（合同冻结口径）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `statute_id` | str(短) | 是 | 法条标识 |
| `law_name` | str(短) | 是 | 法名 |
| `article_no` | str(短) | 否 | 条号 |
| `statute_anchors` | list[{text_id, law_name?, article_no?, anchor_type?}] | 是 | **必填非空**，每条至少 `text_id`（指向法条语料） |
| `article_text` | str | 否 | 条文文本：只来自法条语料、带 `text_id` 锚点，**不得由模型生成** |
| `source_corpus` | str(短) | 否 | 法条语料来源（如 `judge_law_corpus`） |
| `effective_status` | str(短) | 否 | 时效状态（如 `current`），结构化非正文 |
| `related_case_refs` | list[CandidateRef] | 否 | 法条→类案互跳（CandidateRef 同款，零裁判正文） |

`model_config = extra="forbid"`：非白名单 / 正文型 / 模型生成条文型键在模型层即被拒绝。

**StatuteAnchorRef 白名单**：`text_id`（必填非空）/ `law_name` / `article_no` / `anchor_type`。

**合同变更登记位置**：文档 16 §4.1.1 + 文档 17 §3.5。本步为「合同变更」，非擅自加字段。

## 4. 红线断言口径（fail-closed）

1. **条文必锚定语料、不得由模型生成**：`statute_anchors` 必填非空、每条至少 `text_id`；缺锚点 / 锚点不完整即 `ContractViolationError`（「无锚点不展示」）。`article_text` 仅作可核验引用展示，模型不杜撰/改写/续写。
2. **拒绝裁判正文型键**：`full_text` / `content` / `chunk_text` / `raw_case` / `raw_query` / `judgment_text` / `body` 等（复用 E-1 `FORBIDDEN_BODY_KEYS`）。
3. **拒绝富展示型键**：`summary` / `summary_text` / `highlights` / `highlight_text` / `matched_text` / `holding_summary`（与 E3 同口径）。
4. **拒绝 PII 型键**：复用 E4 `is_forbidden_pii_key`（name / id_card / phone / address / ...）。
5. **拒绝模型生成条文型键**：`generated_article` / `generated_text` / `llm_text` / `ai_generated_article` / `paraphrased_article` / `rewritten_article` / `synthesized_article` / ...
6. **异常消息不回显键值**：只暴露键名 / 结构性原因，避免正文 / PII 进入异常或日志。
7. **纯函数不改输入**：`sanitize_statute_ref` / `assert_statute_anchored` 对入参无副作用。

## 5. 法条 ↔ 类案互跳口径

- `StatuteRef.related_case_refs` = `CandidateRef[]`：白名单七字段、**必有 `source_anchors`**（每条至少 `case_id` + `source_chunk_id`）、**零裁判正文**。
- 互跳类案引用即便混入正文型键也 fail-closed 拒绝（不静默搬运裁判正文）。
- **CandidateRef 字段未改**：仍是 E-1 冻结七字段；互跳由服务层（E5-3+）关联，契约不互相内嵌对侧正文。

## 立即回退条件

如未来另行授权做内部受控灰度，任一条件触发即立即回退到 baseline：

| 类别 | 条件 |
| --- | --- |
| 开关一致性 | `ENABLE_WEIGHTED_RERANK=false` 但返回 `score_mode=weighted_rerank`；或受控开启后无法用结构化字段区分路径 |
| schema | `/api/search` 顶层字段或前端依赖的结果项字段缺失 |
| 稳定性 | 5xx 错误率 `>= 1%` 持续 30 分钟 |
| 性能 | warm P95 `>= 3000ms`，或 smoke 出现明显恶化且无非阻断解释 |
| 回滚 | rollback drill 失败、恢复 `>= 60s`、或需要重建向量索引 |
| 精准度 | `P@5 < baseline`、`NDCG@10 <= baseline`、或 `Top10 < 0.60` |
| 回归 | before -> after `REGRESSED > 0`、after vs baseline `REGRESSED > 0`、或 `METRIC_REGRESSION > 0` |
| 前端 | 搜索页白屏、主输入框缺失、提交按钮缺失、或出现阻断渲染错误 |
| 隐私 | 日志或产物出现原始 query、案情正文、候选正文或 chunk 正文 |

回退方式：设置或保持 `ENABLE_WEIGHTED_RERANK=false`，重启或重新装载 API，验证 `/health.feature_flags.ENABLE_WEIGHTED_RERANK=false` 与 `/api/search.results[].score_breakdown.score_mode=base_retrieval`。本轮 rollback drill 显示 weighted rerank 场景恢复 `7ms`，无需重建向量索引。

## 4. 导出格式与内容

格式：`markdown`（默认）、`text`（纯文本）。

内容白名单（100% 来自已收敛 `DraftDescriptor`）：
- `structure_skeleton`：段落标题有序清单（去空白/丢空项；仍是标题，**不含段落正文**）。
- `candidate_refs`：CandidateRef 白名单元数据（案号/法院/审级/案由/裁判日期）+ 来源锚点 `case_id#chunk_id`。
- `statute_refs`：StatuteRef 白名单元数据（法律名称/条号）+ 来源锚点 `text_id`；**显式不取 `article_text`**（即便后端回填语料条文，导出也只携带元数据 + 锚点，回法条页核验条文）。
- `note` / `tag`：用户自填短字段。

## 5. 免责头文案要点（强制注入、不可关闭、不含个案结论）

`DRAFT_EXPORT_DISCLAIMER_LINES` 在 `generateDraftMarkdown` / `generateDraftText` 头部无条件注入（无开关参数）：
1. 本文件仅含结构骨架(标题) + 引用元数据 + 来源锚点 + 用户备注，不含裁判文书正文 / 原始案情 / 起草正文。
2. 数据覆盖说明：底层语料以刑事裁判文书与相关法条为主，为检索与人工筛选的阶段性沉淀，可能存在未覆盖、不代表完整检索。
3. 不承诺检索完整性，不提供胜诉 / 败诉等诉讼结果判断、裁判结果预测及确定性法律结论；本文件为检索沉淀骨架，不构成法律意见。
4. 使用前请结合权威数据库与原始裁判文书 / 现行有效法条人工复核，自行判断相关性与适用性。

免责文案自身经 `containsForbiddenExportPhrase` 校验不含绝对话术（测试断言）。

## 6. 导出边界证明（无正文 / 无结论）

- **无裁判正文**：只读 `DraftDescriptorView` 白名单字段；`judgment_text/chunk_text/summary/highlight/raw_case/raw_query/draft_body/generated_text` grep 0 命中。
- **无条文正文**：`collectStatuteRows` 不取 `ref.article_text`；`grep .article_text` 数据路径 0 命中（仅 2 处注释声明红线）。测试断言虚构条文正文不出现在导出。
- **无胜负 / 结论**：`win_probability/verdict` 0 命中；胜诉/败诉/胜率/必然胜败 仅出现在免责头否定式表述、`FORBIDDEN_EXPORT_PHRASES` 黑名单、注释中，数据生成区 0 命中（测试剔除免责头后扫描）。
- **骨架仅标题**：`structure_skeleton` 导出为有序标题清单，无段落正文。
- **无锚点丢弃**：`collectCandidateRows/collectStatuteRows` 先 `filter(hasAnchor)`，无锚点引用不进入导出（测试断言无锚点 case_b / 某法不出现）。
- **不重新生成**：导出 100% 搬运已收敛字段，零文本生成 / 零补全 / 零结论。

## 7. 鉴权 / 降级（前端落点说明）

本步未新增后端导出端点，故无后端鉴权改动；导出基于前端已取（已经过 E6-2 登录 + gated + 租户隔离返回的）`DraftDescriptor` 在浏览器本地生成。`ENABLE_DRAFTING=false` 时前端入口三重门控不渲染（E6-3 既有），列表为空亦无可导出对象。

导出失败安全降级（`exportDraft` 全程 try/catch，绝不抛出）：
- `draft=null` → `failed/draft_not_found`；空骨架 → `empty/empty_skeleton`；
- 下载器抛错 → `degraded/download_failed`（内容仍返回）；无下载器 → `degraded/download_unavailable`。
- UI 仅 `setExportNotice` 给安全提示，不影响保存 / 列表 / 检索主链路。

日志 `buildDraftExportLog` 只含 `format/status/reason_code` + 三类计数，不含正文/标题/案号/note/tag/draft_id（测试断言序列化后均不命中）。

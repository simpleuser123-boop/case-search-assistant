## Mapping Policy

`data/eval/term_mappings.json` now contains `30` versioned mappings. `21` are default-enabled and `9` are experimental/disabled by default.

High confidence does not automatically mean weighting. Default path is conservative:

- High-confidence default mappings mostly create recall/query variants only.
- Only `ordinary_theft_colloquial` has `use_for_weighting=true`, because it is narrow and low-noise.
- No broad `case_cause_hint` is injected into the default path unless a rule explicitly sets `use_case_cause_hint=true`; current default mappings do not use that.
- Low-confidence mappings are recall-only and never enter `legal_elements`, confidence weighting, or case-cause boost.

## Query Safety

The local mapping path preserves the original cleaned query as `queries[0]`. Every local variant appends mapped terms to the original query instead of replacing it.

Rewrite validation was tightened so variants must preserve strict fact constraints when present, including numeric amounts, dates, time markers, and important responsibility/result markers. If an LLM rewrite drops those constraints, the service degrades to the safe fallback plan. Local tests cover preservation of time/count/action constraints without logging raw request text.

## Default Vs Controlled Experiment

Default path:

- `QueryProcessingService.process()` always applies local mappings before LLM rewrite.
- If `ENABLE_QUERY_REWRITE=false`, local query variants still participate in the fallback plan.
- High-confidence default mappings are recall-only unless `use_for_weighting=true`.
- Low-confidence default mappings are recall-only.
- Experimental mappings are excluded unless `include_experimental=true` is explicitly used outside the main service path.

Controlled experiment only:

- Disabled/experimental mappings listed above.
- Any future `use_case_cause_hint=true` mapping.
- Any broad high-confidence mapping that touches large crime families or ambiguous conduct.

Rollback boundary:

- Disable a single mapping by setting `default_enabled=false` in `data/eval/term_mappings.json`.
- Roll back the whole local mapping path by removing `apply_term_mappings()` use from `apps/api/app/query_processing/service.py` and restoring `QueryPlan` to the previous fields.
- Regenerate the mapping table through `apps/api/app/eval/prepare_lecardv2_eval.py`; it now sources the same versioned default catalog to avoid overwriting this table with stale mappings.

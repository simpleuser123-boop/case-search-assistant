| Component | Weight | Source |
|---|---:|---|
| `vector_similarity` | `0.55` | `vector_score`; fallback to BM25/fallback/base score when vector is absent |
| `legal_element_overlap` | `0.20` | query plan legal elements against candidate fact text |
| `case_cause_match` | `0.10` | query plan case-cause hint against candidate metadata/title |
| `key_paragraph_match` | `0.10` | key paragraph metadata or paragraph prefix |
| `authority_signal` | `0.05` | court level, trial level, judgment year |

Observed issue:

| Signal | Finding |
|---|---|
| `vector_similarity` | In BM25-pool mode this can be normalized fallback/BM25 score, so it is not enough by itself to prove fact similarity. |
| `legal_element_overlap` | Top3 non-relevant candidates had no legal overlap signal in the inspected product-local runs. |
| `case_cause_match` | Top3 non-relevant candidates had `case_cause_match=0`; direct case-cause overweight was not observed, but a low-vector case-cause-only cap was added defensively. |
| `key_paragraph_match` | Before repair, Top3 non-relevant candidates were dominated by raw key paragraph hits: product-chain `71/71`, BM25-pool `71/72`. |
| `authority_signal` | Before repair, Top3 non-relevant candidates all had authority signal: product-chain `71/71`, BM25-pool `72/72`. |

## Repair

Implemented a guarded fusion path in `apps/api/app/rerank/service.py`:

- Keep raw feature scores in `score_breakdown` for explainability.
- Add effective feature scores: `effective_legal_element_overlap`, `effective_case_cause_match`, `effective_key_paragraph_match`, `effective_authority_signal`.
- If no legal-element or effective case-cause support exists, use `base_retrieval_score` as `final_score` and mark `final_score_source=base_retrieval_guard`.
- In that no-fact-support path, key paragraph and authority scores remain visible as raw signals but are set to `0` in effective scoring.
- If case-cause is the only support and vector similarity is below `0.70`, cap effective case-cause at `0.25`.
- Add `fusion_guards` to explain which protection rule fired.

Product eval rows now include raw/effective feature scores plus `final_score_source`, `fusion_guards`, `base_retrieval_score`, `raw_weighted_score`, and `weighted_score`. This is for evaluation evidence only.

Rollback path:

- Runtime rollback remains `ENABLE_WEIGHTED_RERANK=false`.
- Code rollback is limited to reverting the guard/effective-score changes in `apps/api/app/rerank/service.py` and the report-field additions in `apps/api/app/eval/product_eval.py`.
- Baseline comparison files are preserved as `m1.2-product-eval-*-step55-before-20260609.json`.

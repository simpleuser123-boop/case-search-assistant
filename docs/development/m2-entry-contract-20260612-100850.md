## Forbidden Fields And Behaviors

Forbidden in development reports, JSON artifacts, logs, and events:

- original query body.
- case fact body.
- candidate body.
- chunk body.
- unmasked personal data.
- unanchored generated legal explanation.

Forbidden behavior:

- changing online ranking, source selection, recall, or rerank default behavior in M2-1.
- enabling `ENABLE_WEIGHTED_RERANK` by default.
- changing `.env.example` weighted rerank default to `true`.
- changing `ENABLE_EXPANDED_SEARCH` to default `true` without a gate.
- using qrels, labels, relevance, query id, or case id in runtime ranking or source selection.
- deleting bad samples, manually editing ranks, modifying qrels/labels, or editing historical evaluation artifacts.
- writing feedback directly into current ranking weights.
- promising "already exhaustive", "guaranteed no omissions", or any unverifiable recall rate.

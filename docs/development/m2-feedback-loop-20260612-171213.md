## feedback_event schema

后端新增 `FeedbackEventRequest`，只接受以下字段：

| 字段 | 类型 / 枚举 | 说明 |
| --- | --- | --- |
| `event_type` | `result_feedback` | 固定反馈事件类型 |
| `session_hash` | hash | 搜索会话 hash |
| `query_hash` | hash | 本次检索输入 hash |
| `case_id_hash` | hash | 被反馈案例 ID hash |
| `rank` | integer `1..1000` | 结果或候选在当前分组中的展示位置 |
| `feedback_value` | `relevant` / `not_relevant` / `cleared` | 相关、不相关、撤销 |
| `search_mode` | `standard` / `expanded` | 标准检索或扩展检索 |
| `confidence_level` | `high` / `medium` / `low` | 运行时置信度层级 |

hash 字段格式只接受安全 hash 口径：`sha256_*` 或 `fnv1a_*`。

## 脱敏字段说明

事件存储使用 allowlist 重建记录，只保存：

- `event_type`
- `session_hash`
- `query_hash`
- `case_id_hash`
- `rank`
- `feedback_value`
- `search_mode`
- `confidence_level`

前端发送前在内存中计算 `session_hash`、`query_hash`、`case_id_hash`，不会把原始输入、案例 ID 或正文放进请求体。反馈状态只存在当前 React 页面内存中，新检索会清空。

## 禁止字段说明

反馈接口、事件存储、开发报告和 JSON 产物均禁止保存：

- 原始 query
- 原始案情
- 候选正文
- chunk 正文
- 裁判文书正文
- 用户自由文本理由
- 可反推出原始案情的长片段或上下文

后端 schema 使用 `extra=forbid`，正文型字段或自由文本理由字段进入请求时返回 `VALIDATION_ERROR`，不会写入事件存储，也不会把字段值写入日志或响应。

## 后端接口行为

新增接口：

- `POST /api/feedback`

行为：

- 只接收固定脱敏字段。
- 校验 `event_type`、`feedback_value`、`search_mode`、`confidence_level` 枚举值。
- 校验 `rank` 和 hash 字段格式。
- 写入 `FeedbackEventStore` 前按 `FEEDBACK_EVENT_FIELDS` allowlist 重建记录。
- 存储失败时返回 `accepted=true`、`stored=false`、`degraded=true` 和 reason code `FEEDBACK_EVENT_STORAGE_UNAVAILABLE`，不影响检索流程。
- 日志只记录 hash、rank、枚举值、reason code 和 timings，不记录正文或自由文本。

事件存储：

- 默认 JSONL 路径：`data/logs/feedback-events.jsonl`。
- 每行只包含上述八个脱敏字段。
- focused tests 使用内存路径验证 allowlist 行为，避免写入正文型临时产物。

## 前端交互行为

前端新增：

- 每条主结果卡片展示“相关 / 不相关”按钮。
- 每条低置信度候选卡片展示“相关 / 不相关”按钮。
- 点击相同按钮会发送 `cleared`，并撤销本地标记。
- 点击另一个按钮会发送新的反馈值，并修正本地标记。
- 反馈请求失败只显示“反馈未记录”，不打断检索、详情或扩展检索。
- 可见文案只表达“已标记相关 / 已标记不相关 / 反馈未记录”，不出现“系统已学习”“排序已修正”“下次自动优化”等暗示。

前端发送字段：

- `event_type`
- `session_hash`
- `query_hash`
- `case_id_hash`
- `rank`
- `feedback_value`
- `search_mode`
- `confidence_level`

前端不写入 `localStorage`、`sessionStorage` 或 `IndexedDB`。

## feedback 不影响排序的证据

代码边界：

- `POST /api/feedback` 独立于 `/api/search` 和 `/api/search/expand`。
- feedback API 不 import 或调用 retrieval、rerank、source selection。
- feedback 事件不写入 rerank 权重，不写入 qrels、label 或历史评测结果。
- 前端 feedback 点击不触发重新检索，不调用扩展检索，不重排当前数组。

测试证据：

- `test_feedback_does_not_enter_rerank_or_change_same_search_order` 在两次标准检索之间提交 feedback，前后结果顺序一致。
- 同一测试记录 retrieval 调用均为 `include_relaxed_recall=false`。
- 同一测试记录 rerank 只由两次搜索触发，feedback 请求不触发 rerank。
- focused tests 验证 feedback 存储只有脱敏字段。

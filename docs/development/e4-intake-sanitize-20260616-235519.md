## 3. 脱敏规则覆盖项（移除 / 占位，非截断保留）

| 类别 | 占位符 | 识别规则 |
| --- | --- | --- |
| 姓名 | `[姓名]` | 角色标签/关键词定位 + 2-3字边界规则；全文复述一并占位；无姓氏黑名单 |
| 身份证 | `[身份证]` | 18 位、末位可 X/x，前后非数字 |
| 统一社会信用代码 | `[统一社会信用代码]` | 18 位字母数字、至少含一个大写字母（与纯数字身份证区分） |
| 手机号 | `[手机号]` | 1[3-9] + 共 11 位，前后非数字 |
| 银行卡号 | `[银行卡号]` | 16-19 位纯数字，前后非数字 |
| 邮箱 | `[邮箱]` | 标准邮箱正则 |
| 车牌 | `[车牌]` | 省份简称 + 字母 + 5-6 位 |
| 住址 | `[住址]` | 省/市/区县 + 路/街/号/室等街道级要素 |

结构化抽取（尽力而为）：`case_cause`（已知案由表优先 -> 罪名剥离涉嫌/被控等前缀 -> 通用纠纷兜底）、`region`（省/直辖市最早命中）、`trial_level_preference`（再审>二审>一审，含别名归一）、`dispute_focus_keywords`（词典去重保序、上限 8）。`query_text` 仅由已脱敏文本拼成（cap 280），过短时用结构化要素兜底。**双保险**：所有字符串输出再脱敏一次。

## 4. 两侧口径一致性

前端 `sanitize.ts` 与后端 `intake_sanitize.py` 逐项对齐：占位符字面量、PII 正则（用显式 lookaround 而非 `\b` 规避中英文边界差异）、姓名 2-3 字边界规则、罪名前缀剥离表、案由/地域/审级/争议焦点词表、`query_text` cap=280、双保险脱敏。共享 fixture（同一份短假案情）在两侧单测中产出一致的 `case_cause=买卖合同纠纷 / region=上海 / trial_level=二审 / 含货款+违约金`，且 0 PII 残留。

---

## 5. 后端防御性第二道闸

`sanitize_intake_profile_payload(payload)`（纯函数，fail-closed）：

1. 键级红线：`sanitize_intake_search_profile`（内含 `assert_no_raw_case_payload`）拒绝正文型键（raw_case/raw_query/full_text/...）与 PII 型键（name/id_card/phone/address/email/...），并做键级白名单（只保留五字段）。
2. 值级脱敏：对保留下来的字符串值再跑 `redact_pii`——即便键名合法、值里仍夹带 PII（前端漏脱敏），也在此移除/占位。
3. 异常消息只暴露键名、绝不回显键值（已有单测断言假手机号不出现在异常字符串中）。

## 灰度观察指标定义

| 指标 | 定义 | 数据来源 | 隐私约束 |
| --- | --- | --- | --- |
| Top10 hit rate | 有至少 1 个相关 case 出现在 Top10 的 query 数 / 有标注 query 数。 | 离线 qrels；灰度期使用内部人工标注样本，不从日志恢复原始案情。 | 只记录 query id、case id、rank、相关性等级。 |
| Precision@5 | Top5 中相关 case 数 / 5，再对 query 取均值。 | 固定 product-local qrels 与回归集。 | 不记录原始 query 或候选正文。 |
| NDCG@10 | 使用 qrels 等级计算 DCG@10 / IDCG@10。 | 固定 product-local qrels 与回归集。 | 只记录指标和 case id。 |
| 二次搜索率 | `search_refine` 事件数 / `search_result_render` 完成数。 | 前端 analytics：`query_session_id`、`input_length`、`previous_result_count`、`refine_count`。 | 不发送原始案情文本。 |
| 无结果率 | `results.length=0` 的搜索完成数 / 搜索完成数。 | `/api/search` 返回与 `search_zero_result` 事件。 | 只记录结果数量、输入长度、会话 id。 |
| 降级触发率 | `degraded=true` 或 `degraded_reasons` 非空的搜索完成数 / 搜索完成数。 | `/api/search.degraded_reasons`、服务端 `search_completed` 日志。 | 只记录原因枚举、次数、input_hash，不记录原文。 |

上线前还必须先冻结 baseline 行为指标：至少 `100` 次内部搜索完成或 `7` 天内部样本，取先满足者。没有该行为 baseline 时，不允许打开新排序做真实灰度。

## 启用条件

仅当以下条件全部满足时，才允许把新排序列为“灰度候选”；即使满足，也只允许进入内部预演，不允许默认开启或对外放量：

1. 默认开关仍为 `ENABLE_WEIGHTED_RERANK=false`，`.env`、`.env.example`、`Settings` 不得把默认改为开启。
2. baseline 回滚演练通过：`ENABLE_WEIGHTED_RERANK=false` 能在不重建向量索引的情况下恢复到 `score_mode=base_retrieval`，恢复耗时 `< 60s`。
3. `/api/search` schema 稳定：baseline 与受控新排序路径均返回 `query_session_id`、`candidates`、`results`、`degraded`、`degraded_reasons`、`retrieval_duration_ms`、`timings`；结果项必要字段保持稳定。
4. 前端搜索页在开关两侧不白屏，主输入框、提交按钮、结果/空态容器可渲染。
5. 固定 product-local 全量评测无 blocked item，`evaluatedQueryCount=25`。
6. `current after` 指标必须同时满足：`Precision@5 >= baseline`，`NDCG@10 > baseline`，`Top10 hit rate >= 0.60`。
7. 固定回归集必须无明确回归：`before->after REGRESSED=0`，`afterVsBaseline REGRESSED=0`。
8. bad case 输出不得出现为了通过门槛而删除样本、放宽 qrels、修改评测口径或伪造指标的变化。
9. 行为监控基线已冻结，并且灰度预演仪表盘能计算二次搜索率、无结果率、降级触发率。

当前最新证据不满足第 6、7 条，因此不允许进入灰度候选。

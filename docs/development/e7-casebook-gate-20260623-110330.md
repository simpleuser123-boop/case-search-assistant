## 守门覆盖清单

1. **只归集不起草**
   - AST / importlib / 运行时三层守门：`casebook` service/router 不调用文本生成，不暴露案件综述正文、裁判正文、起草正文、胜负或结论字段。
2. **聚合只依赖内核契约**
   - `casebook` 只允许从 `app.kernel`、`app.kernel.guardrails`、`app.kernel.identity` 公开面消费契约与租户上下文，不 import `app.intake` / `app.statute` / `app.drafting`。
3. **引用必带锚点**
   - `CaseFolder.candidate_refs` 与 `draft_descriptors` 内部引用必须有 `source_anchors` / `statute_anchors`；无锚点项运行时 fail-closed 丢弃。
4. **持久层零正文**
   - `CaseFolderRow` 列集合与写入白名单锁死；落库 JSON 不含裁判正文、候选/chunk 正文、起草正文、原始案情；`search_profile_summary` 仅保留脱敏白名单子集。
5. **默认私有 + 对象级鉴权**
   - `visibility` 默认 `private`；仅 `private|team` 两级；private/team/跨租户矩阵完整覆盖；越权 404；仅 owner 可写。
6. **导出守门**
   - 若 E7 后续落导出，则必须带免责头、不得含正文/胜负/结论；当前实现为 absent-or-guarded，守门通过。
7. **casebook 不直连检索底层**
   - AST 断言不 import `retrieval` / `rerank` / `summary` / `query_processing`，也不深引 `app.kernel.rag.*` 底层实现。
8. **产品包互不 import**
   - `casebook` 不 import `intake/statute/drafting`；它们也不 import `casebook`；`app.kernel` 不反向 import `casebook`。
9. **CandidateRef / StatuteRef / DraftDescriptor 仍零正文**
   - 被 `casebook` 引用后字段集合不被增删，继续沿用既有白名单；不新增正文字段。
10. **E8 反哺仍不存在**
    - `ENABLE_ECOSYSTEM` 默认 `false`；`casebook` 可执行路径不存在反馈排序、回流 rerank 或聚合反哺运行时路径。
